Click Calendar Project

Click Calendar is a port of the DateField found in pre Apache Click releases. The DateField had to be refactored and the DHTML popup Calendar removed because LGPL libraries cannot be bundled with Apache Software Foundation distribution.

Please note Click versions 1.5.x and below shipped the DHTML popup Calendar so there is no need to use this library is pre Apache releases.

Only consider this library when using an Apache Click release.

Project structure

The project consists of the Calendar library and an example webapplication.

Following modules are available:

  • [calendar] - the calendar library
  • [calendar-examples] - example usage for the calendar

Other directories:

  • [dist] - build distribution directory
  • [documentation] - all documentation (javadocs too)
  • [lib] - libraries downloaded automatically with get-deps task.

Runtime Dependencies

Click Calendar has the following runtime dependencies

  • click-2.3.0.jar or greater
  • click-extras-2.3.0.jar or greater

Compile time Dependencies

When compiling the project, the following JARs need to be present in the [lib] directory: (see Step 1. in the Building section below on how to automatically download the libraries)

  • click-2.3.0.jar or greater
  • click-extras-2.3.0.jar or greater
  • servlet-api-2.3.jar

Building

To build Click Calendar, execute the following steps:

  1. get all dependencies (and download manually those jars not found in Maven repo), by running:

    ant get-deps

    If you are behind a firewall set your proxy configuration in build.properties and instead run:

    ant get-deps-proxy

  2. build all modules, controls sets, examples, by running:

    ant build-all

  3. (optional) build source packages

    ant build-sources

  4. (optional) build the entire distribution

    ant build-distribution

Changelog
=========

Version 1.3.0 - 2011-03-28
--------------------------

 * Fixed bug where the default date format pattern could not be overridden
   from click-page.properties. With this fix it is easy to define a new
   default date format pattern.

Version 1.2.0 - 2010-05-22
--------------------------

* Added support for min/max year range

* Migrated controls to import resources through Elements and the
  getHeadElements method

* Aligned Calendar with the Servlet 3.0 spec to deploy resources from the directory
  'META-INF/resources' instead of 'META-INF/web'. This release is compatible
  with Click 2.1.0 and up.

Version 1.0.2 - 2009-07-18
--------------------------

* Fixed bug that disabled keyboard input when calendar popup was destroyed.

Version 1.0.1 - 2009-06-15
--------------------------

* Changed the calendar setup to destroy the calendar instance when the popup is closed.
  This fixes a memory leak where calendar instances are not cleaned up [CLK-499].

Version 1.0 - 2009-02-15
------------------------

* Initial release, calendar ported to Apache Click
